## DoNotDev Framework Context (Only use when researching DoNotDev-specific topics)

DoNotDev is a production-ready TypeScript monorepo framework.

## Framework Context

**Architecture:**
- Unified multi-platform (Vite SPA + Next.js SSR)
- Backend flexibility (Firebase + Vercel)
- 11 published packages (8 private via GitHub Packages, 3 public)

**Core Features:**
- Authentication & authorization (@donotdev/auth)
- Stripe billing integration (@donotdev/billing)
- OAuth providers (@donotdev/oauth)
- CMS integration (@donotdev/cms)
- CRUD abstraction layer (@donotdev/crud)
- Schema validation (Valibot - migration complete)
- Feature detection & lazy loading
- Store singleton factory pattern
- Virtual modules for routes/themes/i18n

**Tech Stack:**
- TypeScript (strict mode)
- React 19 (modern patterns)
- Zustand (state management)
- TailwindCSS (styling)
- Vite + Next.js (bundling)
- Bun (package manager + runtime)

**Critical Patterns:**
- Package hierarchy (strict dependency direction)
- Store singleton via createDoNotDevStore()
- Configuration priority: localStorage > props > _DNDEV_CONFIG_ > defaults
- Feature hooks: use{Feature}() with isEnabled check
- Platform abstraction: shared logic + platform wrappers

**Current Status:**
- 3 apps in development (demo live at donotdev.com)
- Framework being sold ($2k lifetime, GitHub team access)
- Valibot migration complete (painless)

When researching, focus on:
- Production-ready patterns (not experimental)
- Type safety and DX
- Performance optimization
- Best practices for framework/library development
- Common pitfalls to avoid
